# 01 — معمارية المشروع | Architecture

## 1. نظرة عليا

```
                          ┌─────────────────────────────┐
        المتصفح (Client)   │   Next.js 14 (App Router)    │
   ┌───────────────────┐   │                             │
   │ واجهة عربية RTL    │◀──┤  Server Components (SEO)    │
   │ Framer / GSAP /3D  │   │  Client Components (تفاعل)  │
   │ Zustand (سلة...)   │   │  Route Handlers (/api)      │
   └─────────┬─────────┘   └───────────┬─────────────────┘
             │                         │
             │ (مجاني، داخل المتصفح)    │ service-role (خادم فقط)
             ▼                         ▼
   ┌───────────────────┐     ┌─────────────────────────┐
   │ Canvas Visualizer │     │  Supabase                │
   │ (لا API / مجاني)  │     │  Postgres + Auth + Storage│
   └───────────────────┘     └─────────────────────────┘
             │                         ▲
             ▼                         │
   ┌───────────────────┐     ┌─────────────────────────┐
   │ WhatsApp (الطلبات)│     │ AI Service Layer (modular)│
   └───────────────────┘     │ browser | replicate | ... │
                             └─────────────────────────┘
```

## 2. مبدأ التصميم: «يعمل فوراً، يتوسّع تدريجياً»

- **الموقع العام** (الرئيسية، المنتجات، الفئات، المدوّنة…) يقرأ من **بيانات تجريبية** في `src/lib/data/*`. لا يحتاج Supabase ليعمل ويظهر بشكل كامل وفخم.
- **لوحة التحكم والطلبات** تستخدم **Supabase** عند توفّر مفاتيح البيئة، وإلا تعمل بوضع «معاينة» آمن.
- **الطلبات** تُرسَل عبر **واتساب** افتراضياً (لا تحتاج خادم)، مع مسار `/api/orders` لحفظها في Supabase عند الرغبة.
- **معاينة الستائر** تتم بالكامل في المتصفح (Canvas) = مجانية وسريعة وخاصة، مع طبقة خدمة جاهزة لاستبدالها بمزوّد AI مدفوع لاحقاً.

## 3. الطبقات

| الطبقة | المسار | الوصف |
|---|---|---|
| العرض/المسارات | `src/app/**` | صفحات Server (للـSEO) تغلّف مكوّنات Client للتفاعل |
| المكوّنات | `src/components/**` | `home, product, visualizer, admin, layout, ui, three, art, seo` |
| منطق الأعمال | `src/lib/**` | ثوابت، i18n، بيانات، Supabase، طبقة AI، حاسبة، بناء طلب واتساب |
| الحالة | `src/store/**` | Zustand: `cart, wishlist, compare` (Persisted) |
| البيانات | `supabase/*.sql` | المخطط والبيانات الأولية |

## 4. تدفّق البيانات (أمثلة)

**عرض منتج:** `app/products/[slug]/page.tsx` (Server, ميتاداتا + JSON-LD) → `ProductDetail` (Client) → يقرأ من `lib/data/products` → تفاعلات (لون/سلة) عبر Zustand.

**إتمام طلب:** السلة (Zustand) → صفحة `/cart` → نموذج توصيل → `buildOrderMessage()` → رابط واتساب. (اختياري) `POST /api/orders` → `service-role` → جدول `orders`.

**معاينة AI:** رفع صورة → `lib/visualizer/draw.ts` يرسم الستارة على Canvas فوق الصورة → حفظ/مقارنة/طلب. طبقة `lib/ai/service.ts` متاحة للتبديل لمزوّد خارجي.

**إدارة:** `/admin/*` محميّة بـ `middleware.ts` (جلسة Supabase) → مكوّنات Client تقرأ/تكتب عبر `@supabase/ssr`.

## 5. الثنائية اللغوية (i18n)

- الافتراضي **عربي RTL**. التبديل عبر `LocaleProvider` (Context) يغيّر `dir/lang` ويعيد ترجمة الواجهة من `lib/i18n/dictionary.ts`.
- المحتوى (منتجات/فئات/مقالات) ثنائي الحقول (`*_ar` / `*_en`).
- النبرة التسويقية **بلهجة عراقية** دافئة.

## 6. الأداء

- مكوّنات الخادم للمحتوى الثابت + توليد ثابت (`generateStaticParams`) للمنتجات والفئات والمقالات.
- استيراد 3D ديناميكياً (`ssr:false`) مع fallback، وتحديد `dpr` للأداء.
- `next/font` لتحميل الخطوط بكفاءة، و`optimizePackageImports`، واحترام `prefers-reduced-motion`.
- لا صور خارجية افتراضياً (رسومات SVG إجرائية) ⇒ لا انتظار شبكة ولا صور مكسورة.

## 7. نقاط التوسعة

- مزوّد AI مدفوع: أضِف ملفاً في `lib/ai/providers/*` وفعّله عبر `AI_PROVIDER`.
- مصدر المنتجات: استبدل قراءة `lib/data` بجلب من Supabase عند الجاهزية (نفس الأنواع).
- الدفع الإلكتروني (ماستر كارد): أضِف Route Handler لبوّابة الدفع + حالة طلب «مدفوع».
